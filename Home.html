Cyber Security in Connected Vehicle Radio Networks

Objective: Security for Autonomous Vehicles

There is a phenomenal burst of interests in Connected Vehicles (CV) data, devices, infrastructures, services, and participants. Despite the potential benefits of CVs, significant security challenges remain to be addressed before widespread deployment of CVs in Intelligent Transportation Systems (ITS). As vehicles become increasingly connected, they face a growing range of cyber security vulnerabilities. In two classes of approaches, prevention-based (such as authentication) and detection-based (such as intrusion detection), can be used to protect wireless network

Security

Network security refers to the protection of access to files and directories in a network system against hacking, misuse and unauthorized changes to the system

Lack of security leaves a system vulnerable to attack

Communication

Connected Vehicles use a form of communication called V2X which contains BSM (Basic Safety Message) information

BSM contains information about the vehicle for example speed, direction, GPS and etc. The BSM information are exchanged with different vehicles which allow the vehicle to be more self aware.

  • Vehicle-To-Pedestrian [V2P]
  • Vehicle-To-Network [V2N]
  • Vehicle-To-Vehicle [V2V]
  • Vehicle-To-Infrastructure [V2I]
